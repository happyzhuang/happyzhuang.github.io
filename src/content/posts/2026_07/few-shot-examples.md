# few-shot 示例

few-shot 指在 Prompt 中给模型几个输入和输出示例，让模型模仿示例完成新任务。它适合格式、风格和判断标准需要稳定的场景。

## 为什么示例有用

有时你用文字描述规则，模型仍然理解不准。给一个示例后，模型更容易学到：

- 输出格式。
- 内容粒度。
- 语气风格。
- 判断标准。
- 不应该输出什么。

## 基本模板

```text
请按示例格式完成任务。

示例 1：
输入：解释 Embedding
输出：
- 一句话定义：Embedding 是把文本转换成向量表示的方法。
- 为什么有用：它让机器可以计算语义相似度。
- 最小例子：用向量检索找到和问题相似的文档。

现在处理：
输入：解释 RAG
输出：
```

## 什么时候用 few-shot

- 批量生成同一种内容。
- 输出格式要求非常固定。
- 需要模仿某种写作风格。
- 分类标准难以用规则说清。
- 模型总是答偏，需要示例校准。

## 注意事项

- 示例要短而典型。
- 示例不要和真实任务矛盾。
- 示例过多会增加 token 成本。
- 示例质量会直接影响输出质量。

## 最小练习

先让模型不看示例解释一个概念。再给它一个示例，让它按示例格式解释同一个概念。比较两次输出是否更稳定。
