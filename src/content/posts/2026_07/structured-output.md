# 结构化输出

如果你只是和模型聊天，自然语言回答就够了。但如果你要把模型接入应用，就需要结构化输出。

## 什么是结构化输出

结构化输出指模型按固定格式返回结果，例如：

- 列表。
- 表格。
- JSON。
- 固定字段。
- 按严重程度排序的问题清单。

这样程序或用户才能稳定使用结果。

## Markdown 表格示例

```text
请把下面内容整理成表格，字段包括：
任务、负责人、截止时间、风险。

内容：
{notes}
```

## JSON 示例

```text
请从下面文本中提取信息，并只输出 JSON：

{
  "name": "",
  "email": "",
  "todo": []
}

文本：
{content}
```

## 注意事项

- 字段越明确，输出越稳定。
- 如果要给程序解析，最好做 JSON 校验。
- 不要假设模型每次都会返回合法 JSON。
- 对重要场景要有重试和兜底。

## 最小练习

把一段会议记录交给模型，让它分别输出：

1. 自然语言总结。
2. Markdown 表格。
3. JSON。

比较三种格式哪个更适合后续使用。
