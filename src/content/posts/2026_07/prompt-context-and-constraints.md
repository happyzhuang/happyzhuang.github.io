# 背景与约束

很多 Prompt 失败，不是因为模型能力不够，而是背景和约束缺失。模型不知道你是谁、要给谁用、有什么限制，就只能自由发挥。

## 背景是什么

背景是和任务有关的上下文，例如：

- 目标读者。
- 项目阶段。
- 技术栈。
- 使用场景。
- 已有资料。
- 你希望避免的问题。

## 约束是什么

约束是模型必须遵守的边界，例如：

- 不超过 300 字。
- 只基于给定材料回答。
- 不要使用营销口吻。
- 输出 Markdown 表格。
- 如果信息不足，请说明缺少什么。

## 示例

弱 Prompt：

```text
帮我写一个 RAG 介绍。
```

更好的 Prompt：

```text
请写一段 RAG 介绍。

背景：读者是刚开始学习大模型应用的中文开发者。
用途：放在学习地图的入门教程中。
约束：
1. 不超过 300 字。
2. 先解释为什么需要 RAG。
3. 不要使用太多论文术语。
4. 最后给一个企业知识库例子。
```

## 最小练习

随便找一个你之前写过的 Prompt，补上三行：

```text
背景：
用途：
约束：
```

再运行一次，对比输出是否更贴近你的真实需求。
