# 闭源模型与开源模型

选择模型时，初学者会遇到两个大方向：闭源模型和开源模型。它们没有绝对好坏，关键看你的任务、预算、隐私要求和部署能力。

## 闭源模型是什么

闭源模型通常由厂商提供 API 或网页产品，模型权重不开放。你通过服务调用模型。

常见特点：

- 接入简单。
- 能力强，更新快。
- 不需要自己部署。
- 费用按量计算。
- 数据和合规要看平台政策。

适合快速做原型、产品验证、对效果要求较高的任务。

## 开源模型是什么

开源模型通常开放权重或推理方式，你可以在本地、服务器或私有云部署。

常见特点：

- 可控性更强。
- 更适合私有化部署。
- 可进行量化、微调和定制。
- 需要硬件和工程维护。
- 效果、速度和成本取决于部署方案。

适合隐私要求高、需要本地运行、需要深度定制的场景。

## 怎么选

| 场景 | 更适合 |
|---|---|
| 快速做 Demo | 闭源 API |
| 效果优先 | 先试闭源强模型 |
| 数据不能出内网 | 开源模型私有化 |
| 成本极敏感且调用量大 | 评估开源部署 |
| 需要微调模型行为 | 开源或支持微调的平台 |

## 初学建议

刚入门时，不建议一上来就纠结“闭源还是开源”。更好的顺序是：

1. 先用网页产品理解能力边界。
2. 再用 API 做最小应用。
3. 再尝试本地模型。
4. 最后根据项目需求做选型。

## 最小练习

选一个任务，例如“总结一篇技术文章”。分别用一个闭源模型和一个本地开源模型尝试，比较：

- 输出质量。
- 响应速度。
- 是否能联网或使用工具。
- 成本和部署难度。
