# ChatGPT / Claude / Gemini

ChatGPT、Claude、Gemini 都是主流大模型产品。对初学者来说，它们最重要的价值不是“谁最强”，而是帮助你观察不同模型在同一个任务上的表现差异。

## 学习时怎么使用它们

你可以把这些产品当成大模型能力观察台：

- 同一个 Prompt，分别发给不同模型。
- 观察输出结构、准确性、风格和追问能力。
- 记录哪个模型更适合你的任务。
- 不要只看一次回答，要多试几个场景。

## 对比维度

| 维度 | 观察什么 |
|---|---|
| 中文表达 | 是否自然、准确、有层次 |
| 代码能力 | 是否能给出可运行代码和解释 |
| 长文理解 | 是否能抓住长文重点 |
| 推理能力 | 是否能分步骤解决问题 |
| 多模态能力 | 是否能看图、读截图、理解图表 |
| 工具生态 | 是否支持文件、搜索、代码执行等 |

## 不要迷信单次体验

模型输出有随机性，网页产品也会持续更新。一次回答好坏不能代表全部能力。更靠谱的方法是建立自己的任务集，比如：

- 解释一个技术概念。
- 审查一段代码。
- 总结一篇文章。
- 根据资料回答问题。
- 生成一个项目计划。

然后用同一组任务做对比。

## 最小练习

把下面 Prompt 分别发给不同模型：

```text
请解释什么是 RAG。
要求：
1. 面向刚学大模型应用的开发者。
2. 用一句话定义。
3. 用 5 步说明流程。
4. 给一个企业知识库例子。
5. 最后列出 3 个常见误区。
```

比较它们的结构、准确性和可读性。
