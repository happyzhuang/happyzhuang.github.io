# LLM, Hello! 网站规划文档

## 1. 网站定位

`llmhello.com` 不再只是一个普通技术博客，而是逐步升级为一个围绕大模型、Agent、Prompt 与人机协作的个人实验型知识站点。

核心定位：

- LLM 时代的个人技术实验室
- 大模型工程实践的学习地图
- Prompt、RAG、Agent 等主题的长期内容档案
- 可交互 AI 小工具和项目日志的展示入口

一句话描述：

> LLM, Hello! 是一个关于大模型、Agent、Prompt 与人机协作的实验笔记。

## 2. 总体策略

当前项目基于 Astro 静态站点，托管在 GitHub Pages。短期不重起新项目，也不整体迁移云服务。

推荐路线：

1. 保留现有 Astro 博客架构。
2. 先扩展信息架构和静态栏目。
3. 再加入不依赖后端的前端互动工具。
4. 后续如需真实 LLM API，再单独增加后端 API 服务。
5. 只有当出现用户系统、数据库、文档上传、私有知识库等复杂需求时，再考虑独立应用。

未来可能的部署形态：

```text
llmhello.com
= GitHub Pages
= Astro 静态内容站 + 前端互动工具

api.llmhello.com
= Cloudflare Workers / Vercel Functions / Railway / Render
= 模型 API 代理、限流、鉴权、日志

app.llmhello.com
= Next.js / SvelteKit / Astro SSR
= 登录、用户知识库、复杂 AI 工具
```

## 3. 信息架构

规划中的一级栏目：

- 首页：站点入口，展示定位、核心入口、最新文章。
- 实验室：可交互工具、流程演示、模型行为观察。
- 学习地图：LLM 学习路线和文章组织框架。
- Prompt 配方：可复用提示词模板和实践说明。
- 项目日志：记录真实项目从想法到上线的过程。
- 归档：保留原有文章归档能力。
- 关于：个人介绍和站点说明。

## 4. 首页规划

首页不再放完整文章列表，而是一个正式的网站门户页。文章列表保留在归档页，首页只保留少量最新更新入口。

首页模块：

- 站点定位：
  `LLM, Hello! 一个关于大模型、Agent、Prompt 与人机协作的实验笔记。`
- 四个核心入口：
  - 学习地图
  - 实验室
  - Prompt 配方
  - 项目日志
- 最新更新：只展示少量文章入口
- 后续可增加：
  - 最新实验
  - 推荐阅读路径
  - 当前正在构建的项目

## 5. 栏目规划

### 5.1 实验室

目标：让网站从“阅读型”变成“体验型”。

优先实验：

- Token 成本估算器
- Prompt 对比器
- RAG 流程可视化
- Agent 工作流演示
- 模型输出观察笔记

第一阶段只做前端工具，不接真实 LLM API。

### 5.2 学习地图

目标：把零散文章组织成长期学习路线。

路线节点：

- LLM 基础
- Prompt Engineering
- Embedding 与 RAG
- Tool Use / Function Calling
- Agent 工程
- 评测与成本
- 应用开发
- 部署与运维

后续动作：

- 把已有文章挂到对应节点。
- 没有文章的节点标记为计划中。
- 每个节点可以链接文章、实验、外部资料或项目日志。

### 5.3 Prompt 配方

目标：形成个人化 Prompt Cookbook。

每个配方建议包含：

- 标题
- 适用场景
- Prompt 模板
- 输入示例
- 输出示例
- 注意事项
- 容易失败的地方

优先配方：

- 结构化技术解释
- 代码审查助手
- 项目日志整理
- 文章大纲生成
- RAG 问答提示词
- Agent 任务拆解提示词

### 5.4 项目日志

目标：记录真实构建过程，而不只是最终结果。

项目日志适合记录：

- 背景和动机
- 技术选型
- 关键决策
- 踩坑和修复
- 上线过程
- 后续计划

优先项目：

- LLM, Hello! 网站升级
- 个人 RAG 知识库
- Agent 工程样板
- Prompt 对比器
- 博客 AI 助手

## 6. 分阶段计划

### 第一阶段：信息架构升级

目标：让网站从普通博客变成 LLM 实验室型知识站点。

内容：

- 修复站点中文文案。
- 扩展导航。
- 新增实验室、学习地图、Prompt 配方、项目日志页面。
- 首页新增核心入口区。
- 保留现有文章列表和归档页。

预计难度：低。

### 第二阶段：轻量互动工具

目标：加入不依赖后端的前端体验。

内容：

- Token 成本估算器。
- Prompt 对比器。
- RAG 流程静态/交互演示。
- Agent 工作流演示。

预计难度：中低。

### 第三阶段：内容体系填充

目标：让新栏目真正变厚。

内容：

- 把已有文章映射到学习地图。
- 为 Prompt 配方补充更多模板。
- 为每个项目日志写第一篇记录。
- 建立标签和分类规范。

预计难度：中低，主要投入在内容生产。

### 第四阶段：真实 AI 能力

目标：引入真实模型调用，但不破坏主站静态架构。

内容：

- 新增独立 API 服务。
- 在线 Prompt Playground。
- 简单文档问答 Demo。
- 博客 AI 助手。
- 限流、费用控制和 API Key 保护。

预计难度：中高。

### 第五阶段：独立应用

触发条件：

- 需要用户登录。
- 需要用户上传文档。
- 需要保存个人历史记录。
- 需要数据库或向量数据库。
- 需要复杂 Agent 后台任务。

建议形态：

- `llmhello.com` 继续作为内容主站。
- `app.llmhello.com` 承载完整 AI 工具平台。
